Rationale What and Why

Our team forced on improving the layout and more user friendly version of the existing Comp1950 Website. We have improved the layout, changed colour scheme, implemented more Responsive Web Design and added new features throughout the Prototype Site. As the users ourselves we wanted to improve students’ experience and interaction with the website by being more interactive such as having a calendar, being able to comment, mobile friendly viewport and improved SEO.

Shis Meta site also includes planned Site Map for visual representation.

Rationale How

RWD: Students usually use public transportation to arrive the class and have plenty of time to scroll down on a website. We have realized the existing website is lacking that option for students so we added @media screen properties and adjusted the max width and min width for different devices like phones or tablets. Pages toggle down to a mobile view when the device or browser capabilities change.

Printer styles added for an easier print out for students who like to study hands on. The styles can be found project2.css file.

JavaScript:We have changed the layout to make it visually more appealing and added Javascript to demonstrate what is being taught The home page includes Javascript as a slider images.

CSS3: We have implemented multiple images throughout the site to improve visually and also demonstration. Our <header> images has background-image and its attributes.

SEO:We have aimed to improve our Search Engine Optimization by improving HTML5 and page semantics. Examples can be <section> tag, added new from attributes such as <placeholder> and type=email.

Advanced CSS :We used CSS reset for more predictable content in most browser and to minimize odd behaviour in rendering. We have used @import to manage multiple CSS files for our normalized css folder and template css folders and minimized versions such as bootstrap.css and font-awesome.min.css

Rationale More Features

Search feature: students can find the information they need easier with keywords While navigating through the current website for Comp 1950 students usually find it difficult to find a specific topic or heading. Search button solves this problem by allowing students search for keywords through the site and displays the results. This feature is only a prototype at this point of the project. If our project would be chosen than this feature would be implemented.

Calendar widget: click on the calendar widget to display what day it is and when the upcoming homework/ quiz/ project is. It saves time and it is all at the same place so it is difficult for the students to miss out on deadlines. This feature is only a prototype at this point of the project. If our project would be chosen than this feature would be implemented.

Lectures pages include:

Highlight “to do”: remind students about homework/project/quizs

Comments: Students can ask questions here to the instructor and the classmates. Lecture focused questions or comments can be added by the students for more interactive learning environment. Students or the instructor can add comments and ask and answer questions. This feature is only a prototype at this point of the project. If our project would be chosen than this feature would be implemented.

Resources: students can find important resources easily. It is more accessible to the students in terms of visuality and navigation.
